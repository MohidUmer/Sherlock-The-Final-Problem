Moriarty has infiltrated the Baker Street relay. Sherlock sends one message. Watson receives another. The archive trusts the very headers a client can shape. Find the disagreement. Poison the truth. Claim the archive.

Case Intelligence

Objective

Poison the shared message cache so an ordinary request receives a privileged archive response with a leaked session token.

Vulnerability

The relay trusts X-Forwarded-Host for privilege decisions while keying the cache on the public Host header—two different trust domains.

Fingerprint

Watch X-BakerStreet-Cache and X-BakerStreet-Cache-Key response headers. They tell you whose response you are actually receiving.

Final Gate

The archive at /api/v2/archive/final demands the admin host context and a valid session token leaked by the poisoned response.

"The message you received was not the one I sent. Observe the seal, not merely the letter—hosts, proxies, and caches all leave fingerprints." — Sherlock Holmes, Baker Street Relay Transmission
Attack Architecture

Five steps.
One poisoned truth.

Observe

Read /api/v2/messages/latest. Notice the cache metadata and the observed host the relay reports.

Diagnose

Inspect /api/v2/relay/diagnostics to understand how cache keys are constructed and where the split trust boundary lies.

Poison

Send X-Forwarded-Host: admin.bakerstreet.local + Cache-Control: no-cache to generate admin content in the public cache bucket.

Trigger

Make a normal request. The cache serves the poisoned admin response—including the leaked archive session token.

Archive

Call /api/v2/archive/final with the admin host context and the leaked session. Claim the flag.
